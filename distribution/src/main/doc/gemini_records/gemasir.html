10.0 ASIR - Array Status Information Record

The Array Status Information Record provides a standard information-passing mechanism between Gemini principle systems. The ASIR records for any Gemini system are expected to be combined into a separate database called the Status Alarm Database (SAD) and loaded into the SAD IOC.

ASIR records combine three important parts of Gemini EPICS status reporting: a status value, status message, and alarms. The status value is read through the INP link, the status message is read through the IMSS link. Alarms are propagated to the SIR record by the input link.

There are three valid output data types: long integer, double precision, and string. Arrays of these datatypes are also supported with the use of the NELM field. A subroutine supplied by the user can be attached to the SNAM field and called during record processing. This subroutine can convert the input, raise additional alarms, or enhance the output message string.

A full description of the ASIR record's function is expected in the FDSC field. This field can hold a 40 character string which is used by the OCS when displaying the ASIR value.

The ASIR and SIR records are nearly identical. The differences are (a) the ability to have an array for the value, and (b) the change in link type of the IMSS field.

10.3 Record Support Routines

10.3.1 init_record

On the first pass, space is allocated for the VAL, SVAL, MVAL, AVAL fields, based upon the type of FTVL field (DBF_STRING, DBF_DOUBLE, or DBF_LONG) and the number of elements specified by NELM. On the second pass, the simulation mode is checked and if true, the simulation mode and input link are set. The address of the process subroutine (SNAM) is found.

10.3.2 process

See the next section.

10.3.3 get_value

This routine fills the values of struct valueDes so that they refer to VAL.

10.3.4 get_precision

This routine retrieves PREC.

10.3.5 get_units

This routine returns EGU.

10.3.6 cvt_dbaddr

This routine converts the VAL field to either string, double precision, or long integer based upon the value of FTVL.

10.3.7 get_alarm_double

This routine sets the following values:
upper_alarm_limit = HIHI
upper_warning_limit = HIGH
lower_warning_limit = LOW
lower_alarm_limit = LOLO

10.4 Record Processing

In simulation mode, the simulation value is fetched from the link. Otherwise the values is fetched from the INP link. The user subroutine (SNAM) is executed and the return value is checked. For return values of zero, the value found through the input message link IMSS is copied to the output message OMSS, the record is time stamped, alarms and monitors are checked, and the forward link is processed.

An alarm is set if the value is outside the preset range limits (HIHI, HIGH, LOW, LOLO) and outside of the hysteresis range (HYST).

Logging and archive monitors are raised on VAL and OMSS if their value changes. For double precision and long integer values of VAL the change must be greater than the hysteresis ranges (MDEL and ADEL).

10.5 Device Support

No device support is available.
